自建memos笔记

memos部署

memos笔记类似于flomo的卡片笔记，可以使用docker一键部署

text
docker pull ghcr.io/usememos/memos:0.22.3

版本可以自行选择，选这个版本是因为这个版本支持ai助理

然后

text
docker run -d --name memos -p 5230:5230 -v ~/.memos/:/var/opt/memos ghcr.io/usememos/memos:0.22.3

最后ip+端口即可访问了，memos的数据存放在 ~/.memos/ 中

n8n部署

n8n是一个低代码平台, 可以通过docker部署，现在简单说下在render上怎么部署

注册成功后：

镜像输入：docker.io/n8nio/n8n:latest

地区选新加坡

数据库继续白嫖 [supabase](https://supabase.com/)

得到

bash
postgresql://postgres.mpxiqrcjwtjqusjktffu:【创建数据库时得到的密码】@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres

最后在reder的环境变量配置如下

  • DB_POSTGRESDB_DATABASE: postgres

  • DB_POSTGRESDB_HOST: aws-0-ap-southeast-1.pooler.supabase.com

  • DB_POSTGRESDB_PASSWORD: 【创建数据库时得到的密码】

  • DB_POSTGRESDB_PORT: 6543

  • DB_POSTGRESDB_SCHEMA: public

  • DB_POSTGRESDB_USER: postgres.mpxiqrcjwtjqusjktffu

  • DB_TYPE: postgresdb

  • WEBHOOK_URL: 部署n8n成功后得到的网址

  • GENERIC_TIMEZONE: Asia/Shanghai

  • N8N_ENCRYPTION_KEY: 自己记得住的任意密码

最后用[uptime](dashboard.uptimerobot.com)保活

AI助理配置

通过reder的地址登陆上n8n

https://n8n.akashio.com/b121c42b66fe4963925baa70de007dd6

网址授权：把url和token配置为对应memos的网址和生成的token

再在最后的Basic LLM节点加上三个，可以选gemmi，openapi啥的

数据备份

这个就看个人吧，比如可以用github，也可以挂载一些网盘啥的

配置 cron job，在 /etc/crontab 中增加自动备份定时任务，例如每天凌晨3点进行备份；

bash
0 03 * * * root sh /root/memos-backup/backup.sh

这个是backup脚本，搞定

text
# memos data directory is ~/.memos/
tar -czf memos-archive.tar.gz ~/.memos/memos_prod.db
message="backup in `date '+%Y%m%d%H%M'`"
git add -A .
git commit -s -m "${message}"
git push
